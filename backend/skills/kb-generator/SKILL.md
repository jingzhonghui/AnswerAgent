# 知识库文件生成规范

当被要求生成知识库文件时，必须遵守以下规范。

## 文件格式

每个知识库文件使用 Markdown 格式，以一级标题 `# 标题` 开头。

可选的 YAML frontmatter：
```yaml
---
title: 文件标题
description: 简短描述
tags: [标签1, 标签2]
---
```

## 推荐章节结构

根据文件类型选择合适的章节：

| 文件类型 | 推荐章节 |
|----------|----------|
| 项目概述 | 项目简介、核心功能、技术栈、快速开始、目录结构 |
| 架构设计 | 整体架构、模块说明、数据流、关键设计决策 |
| API 参考 | 接口列表、请求/响应示例、参数说明、错误码 |
| 安装指南 | 环境要求、安装步骤、配置说明、验证方法 |
| 配置说明 | 配置项列表、默认值、环境变量、示例配置 |
| 开发指南 | 开发环境搭建、代码规范、调试方法、贡献指南 |

## 格式要求

1. **标题层级**: 一级标题 `#` 用于文件标题，二级 `##` 用于主要章节，三级 `###` 用于子章节
2. **代码块**: 使用 ` ```language ` 包裹，指定正确的语言标识
3. **表格**: 用于对比、列举、配置项等结构化信息
4. **链接**: 引用同知识库内其他文件使用相对路径 `[名称](./other-file.md)`
5. **术语**: 首次出现的专业术语使用 **加粗** 强调

## 内容质量

1. **准确性**: 基于源文件内容，不编造信息
2. **完整性**: 代码示例可直接运行，配置示例完整
3. **简洁性**: 避免冗余，用表格替代长段落
4. **一致性**: 同一知识库内术语、格式、风格保持一致
5. **中文优先**: 正文使用中文，技术术语保留英文原文

## 示例

```markdown
# 项目概述

## 项目简介

AnswerAgent 是一个基于本地知识库的智能问答系统。

## 核心功能

| 功能 | 说明 |
|------|------|
| 知识库问答 | 基于本地文件回答问题 |
| 深度思考 | 多步推理探索知识库 |

## 技术栈

- **后端**: Python + FastAPI + DeepAgents
- **前端**: Vue 3 + TypeScript + Vite
- **存储**: SQLite + JSON 文件
```
