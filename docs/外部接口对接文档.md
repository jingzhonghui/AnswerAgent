# AnswerAgent 外部接口对接文档

> 版本 0.1.0 | 面向外部系统开发者

---

## 1. 概述

AnswerAgent 提供免登录的流式问答接口，外部系统可直接通过 HTTP POST 请求调用，无需认证、无需管理会话。

- **接口地址**: `POST http://<host>:8765/api/chat/external/stream`
- **请求格式**: `application/json`
- **响应格式**: `text/event-stream`（SSE，Server-Sent Events）

---

## 2. 快速开始

```bash
curl -X POST http://localhost:8765/api/chat/external/stream \
  -H "Content-Type: application/json" \
  -d '{"message": "你好，请介绍一下你自己"}'
```

响应是 SSE 流，以 `\n\n` 分隔事件：

```
event: kb_matched
data: {"kb_names":["技术文档"]}

event: token
data: {"content":"你好！我是"}

event: token
data: {"content":"AnswerAgent，"}

event: token
data: {"content":"一个基于知识库的智能问答助手。"}

event: done
data: {}
```

---

## 3. 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `message` | string | ✅ | - | 用户消息内容，至少 1 个字符 |
| `mode` | string | ❌ | `"default"` | 聊天模式：`"default"` 或 `"deep"` |
| `history` | array | ❌ | `[]` | 可选历史消息列表，用于多轮对话 |

### 3.1 mode 说明

| 值 | 说明 |
|------|------|
| `"default"` | 标准 LCEL 问答链，快速流式回答 |
| `"deep"` | ReAct Agent 深度思考模式，支持多步推理和工具调用 |

### 3.2 history 格式

`history` 数组中的每项为一个对象：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `role` | string | ✅ | `"user"` 或 `"assistant"` |
| `content` | string | ✅ | 该条消息的完整内容 |

**示例（多轮对话）：**

```json
{
  "message": "那第二个方案呢？",
  "history": [
    {"role": "user", "content": "请帮我分析这三个技术方案的优劣"},
    {"role": "assistant", "content": "好的，以下是分析：\n\n**方案一：微服务架构**\n优点：独立部署、技术栈灵活\n..."},
    {"role": "user", "content": "详细说说方案一的部署流程"},
    {"role": "assistant", "content": "方案一的部署流程如下：\n1. 容器化各服务\n2. 配置 Kubernetes\n..."}
  ]
}
```

## 4. SSE 事件类型

### 4.1 事件一览

| 事件 | 说明 | default 模式 | deep 模式 |
|------|------|:---:|:---:|
| `kb_matched` | 知识库匹配完成 | ✅ | ✅ |
| `files_selected` | 文件选取完成（每个知识库一次） | ✅ | ✅ |
| `token` | 流式回答片段 | ✅ | ✅ |
| `agent_think` | 深度思考推理步骤 | - | ✅ |
| `agent_observe` | 工具执行结果 | - | ✅ |
| `done` | 回答完成 | ✅ | ✅ |
| `error` | 错误信息 | ✅ | ✅ |

### 4.2 事件数据格式

#### kb_matched

```
event: kb_matched
data: {"kb_names": ["知识库A", "知识库B"]}
```

`kb_names` 为空数组 `[]` 表示未匹配到任何知识库，此时走通用问答模式。

#### files_selected

```
event: files_selected
data: {"kb": "知识库A", "files": ["doc/guide.md", "api/reference.md"]}
```

#### token

```
event: token
data: {"content": "这是一段回答内容"}
```

> 注意：每个 token 事件只包含部分内容，调用方需自行拼接完整回答。

#### agent_think（仅 deep 模式）

```
event: agent_think
data: {
  "step": "action",
  "thought": "用户想知道知识库中关于API设计的最佳实践...",
  "tool": "search_knowledge",
  "tool_input": "{'query': 'API设计最佳实践'}"
}
```

#### agent_observe（仅 deep 模式）

```
event: agent_observe
data: {
  "step": "observation",
  "result": "在 doc/api-design.md 中找到相关内容：\n..."
}
```

#### done

```
event: done
data: {}
```

> 外部接口的 `done` 事件返回空对象，不含 `message_id` 和 `title`（不同于内部接口）。

#### error

```
event: error
data: {"message": "错误描述"}
```

## 5. 完整示例

### 5.1 default 模式

**请求：**

```bash
curl -X POST http://localhost:8765/api/chat/external/stream \
  -H "Content-Type: application/json" \
  --no-buffer \
  -d '{"message": "AnswerAgent 如何配置知识库？"}'
```

**响应流：**

```
event: kb_matched
data: {"kb_names":["AnswerAgent文档"]}

event: files_selected
data: {"kb":"AnswerAgent文档","files":["README.md","docs/配置指南.md"]}

event: token
data: {"content":"配置"}

event: token
data: {"content":"知识库"}

event: token
data: {"content":"非常简单，"}

event: token
data: {"content":"只需在"}

event: token
data: {"content":"knowledge"}

event: token
data: {"content":"目录下"}

event: token
data: {"content":"创建子"}

event: token
data: {"content":"目录即可"}

event: token
data: {"content":"。"}

event: done
data: {}
```

### 5.2 deep 模式

**请求：**

```bash
curl -X POST http://localhost:8765/api/chat/external/stream \
  -H "Content-Type: application/json" \
  --no-buffer \
  -d '{"message": "比较微服务和单体架构的优劣", "mode": "deep"}'
```

**响应流：**

```
event: kb_matched
data: {"kb_names":["架构设计"]}

event: files_selected
data: {"kb":"架构设计","files":["microservices.md","monolith.md"]}

event: agent_think
data: {"step":"action","thought":"需要从知识库中检索微服务和单体架构的对比信息","tool":"search_knowledge","tool_input":"{'query': '微服务 单体 架构 对比'}"}

event: agent_observe
data: {"step":"observation","result":"在 microservices.md 中找到：微服务架构将应用拆分为..."}

event: agent_think
data: {"step":"action","thought":"还需要查找单体架构的详细信息","tool":"search_knowledge","tool_input":"{'query': '单体架构 优点 缺点'}"}

event: agent_observe
data: {"step":"observation","result":"在 monolith.md 中找到：单体架构的优势在于..."}

event: token
data: {"content":"根据知识库内容，微服务和单体架构的对比分析如下：\n\n| 维度 | 微服务 | 单体 |\n|------|--------|------|\n| 部署 | 独立部署 | 整体部署 |\n..."}

```

## 6. 错误处理

| 状态码 | 场景 | error 事件示例 |
|--------|------|----------------|
| - | 请求参数校验失败 | `{"message": "请求参数错误: ..."}` |
| - | LLM 配置未设置 | `{"message": "LLM 配置错误: ..."}` |
| - | 知识库路由异常 | `{"message": "服务器内部错误: ..."}` |
| - | 客户端断开连接 | 流中断，无额外事件 |

> 注意：所有错误都通过 SSE `error` 事件返回，HTTP 状态码始终为 200（因为 SSE 连接已建立）。

### 客户端断开

当客户端断开连接时，服务端停止生成，不再发送后续事件。不会保存已生成的部分内容（与内部接口不同）。

---

## 7. 调用建议

### 7.1 超时设置

default 模式的回答通常在 5-30 秒内完成，deep 模式可能需要 30-120 秒。建议客户端设置合理的读取超时（如 180 秒）。

### 7.2 重试策略

由于接口无状态，失败时直接重试即可，无需维护会话状态。

### 7.3 并发控制

接口没有内置速率限制。建议在反向代理层（如 Nginx）配置：

```nginx
# 限制每个 IP 的并发连接数
limit_conn_zone $binary_remote_addr zone=chat:10m;
limit_conn chat 5;

# 限制请求速率
limit_req_zone $binary_remote_addr zone=chat_rate:10m rate=10r/m;
limit_req zone=chat_rate burst=5;
```

### 7.4 历史消息管理

`history` 字段由调用方自行管理。建议每次请求都传入完整的对话历史，不要依赖服务端记忆（服务端不持久化外部接口的对话）。

### 7.5 安全建议

- 接口无认证，建议仅在内网或受信任环境中使用
- 如需对外暴露，请在反向代理层添加 IP 白名单
- 不要在 `history` 中传入敏感信息，因为消息内容会作为 LLM 上下文发送给模型提供商

## 8. 代码示例

### JavaScript (fetch + ReadableStream)

```javascript
async function chat(message, history = []) {
  const response = await fetch('http://localhost:8765/api/chat/external/stream', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ message, history }),
  });

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = '';
  let fullAnswer = '';

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop() || '';

    let eventType = '';
    for (const line of lines) {
      if (line.startsWith('event: ')) {
        eventType = line.slice(7).trim();
      } else if (line.startsWith('data: ')) {
        const data = JSON.parse(line.slice(6));
        if (eventType === 'token') {
          fullAnswer += data.content;
          process.stdout.write(data.content); // 实时输出
        } else if (eventType === 'done') {
          console.log('\n--- 回答完成 ---');
        } else if (eventType === 'error') {
          console.error('错误:', data.message);
        }
      }
    }
  }

  return fullAnswer;
}

// 使用
chat('你好，请介绍一下你自己');
```

### Python (requests + sseclient)

```python
import requests
import json

def chat(message, history=None, mode="default"):
    url = "http://localhost:8765/api/chat/external/stream"
    payload = {
        "message": message,
        "mode": mode,
        "history": history or [],
    }

    response = requests.post(url, json=payload, stream=True)
    full_answer = ""

    for line in response.iter_lines(decode_unicode=True):
        if not line:
            continue

        if line.startswith("event: "):
            event_type = line[7:]
        elif line.startswith("data: "):
            data = json.loads(line[6:])
            if event_type == "token":
                full_answer += data["content"]
                print(data["content"], end="", flush=True)
            elif event_type == "done":
                print("\n--- 回答完成 ---")
            elif event_type == "error":
                print(f"\n错误: {data['message']}")

    return full_answer


# 使用
chat("你好，请介绍一下你自己")

# 多轮对话
chat(
    message="那第二个呢？",
    history=[
        {"role": "user", "content": "分析这三个方案"},
        {"role": "assistant", "content": "方案一：微服务\n方案二：模块化单体\n方案三：Serverless"},
    ]
)
```
